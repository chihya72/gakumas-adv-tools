## 📖 使用指南

### 如何使用这个表格

1. **查询已知表情**：在"已验证表情"中查找确认含义的 index
2. **实验新表情**：从"待测试表情"中选择感兴趣的 index 进行测试
3. **创建组合**：参考"组合表情"设计复杂情感表达

### 实验方法

#### 方法 1: 修改现有文件

```txt
1. 备份 gakumas-data/data/ 中的任意 .txt 文件
2. 找到 [actorfacialoverridemotion ...] 行
3. 修改 "index":数字 为你要测试的值
4. 游戏内观察效果
```

**示例**：
```txt
原始:
"faceModels":[\{"path":"Root_Face","index":20,"value":1.0\}]

修改为（测试 index 8）:
"faceModels":[\{"path":"Root_Face","index":8,"value":1.0\}]
```

#### 方法 2: 创建测试组合

```txt
单个表情:
"faceModels":[\{"path":"Root_Face","index":22,"value":1.0\}]

组合表情:
"faceModels":[\{"path":"Root_Face","index":2,"value":1.0\},\{"path":"Root_Face","index":42,"value":1.0\}]
```

## 📊 统计信息

### 表情分类（基于时长）

- **瞬时动作**（<1.5s）：43, 52, 72, 47, 9, 50 等
- **短暂表情**（1.5-2.5s）：20, 21, 41, 51, 48 等
- **常规表情**（2.5-4.5s）：大部分 index
- **持续表情**（>4.5s）：32, 33, 40

### Value 值模式

- **正值为主**：98% 的使用是 `value=1.0`
- **负值用法**：
  - Index 0: 40% 使用 `-1.0`
  - Index 20: 13% 使用 `-1.0`
  - 推测：负值可能表示"禁用"或"反向"

### 表情系统设计模式

根据已验证的 7 个 index，可以看出表情系统的设计模式：

1. **眼部控制**（Index 20, 22）
   - 20 = 眨眼（动态）
   - 22 = 闭眼（静态）

2. **嘴部表情**（Index 42, 71, 73）
   - 71 = 闭嘴微笑（小）
   - 42 = 闭嘴大笑（大）
   - 73 = 抿嘴（克制）

3. **眉毛/情绪**（Index 2, 3）
   - 3 = 皱眉担忧（柔和）
   - 2 = 皱眉愤怒（强烈）

4. **组合效果**
   - 眼部 + 嘴部 = 复合情感
   - 眉毛 + 嘴部 = 带情绪的笑容
